if: Conditional Components

ftd supports an if expression to decide if the component should be visible or not, based on the arguments, or global variables.

Conditionals When Using Components
-- var dark-mode: true

-- ftd.text: we are in dark mode
if: dark-mode

-- ftd.text: light mode rules
if: not dark-mode

Here we have used the if header to describe the condition in which the component is visible. Since our variable dark-mode is set to true, the first ftd.text, will be visible, and the second will not be.

Conditionals In Components

When defining a component we can also use conditionals:

-- component image:
component: ft.column
$src: string
$caption: optional caption

--- ft.image:
src: ref $src

--- ft.text:
if: $caption is not null
text: ref $caption

Here we have created an image component, with an optional caption. If caption is passed, that is $caption is not null, the ftd.text is rendered.

Supported Conditions

Currently, we support the following conditions:

if: $foo is null

Only show the component when the optional value is missing.

if: $foo is not null

Only show the component when the optional value is passed.

if: $foo

Only show the component if $foo is true. Can only be used if $foo is a boolean.

It can also use a global variable: if: foo (assuming foo is declared earlier as a boolean variable).

if: not $foo

Only show the component if $foo is false, otherwise same as if: $foo.

if: true | if: false

if: false can be used to disable a component without deleting it. if: true is no-op, can be safely deleted.
